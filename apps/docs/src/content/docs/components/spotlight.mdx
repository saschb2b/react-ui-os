`Spotlight` is the keyboard front door. It is self-contained: owns its own open/close state, listens for both Cmd-K and a custom event, and restores the previously focused element when it closes.

<LivePreview src="?demo=spotlight" label="Spotlight opens on load" height="420px" />

## Import

```tsx
import { Spotlight, SPOTLIGHT_OPEN_EVENT } from "@react-ui-os/desktop";
```

## Props

None. Drop the component anywhere inside `<DesktopProvider>` and it works. The default `<Desktop>` already mounts it.

## Opening Spotlight

| Trigger                                                         | Behavior                                                        |
| --------------------------------------------------------------- | --------------------------------------------------------------- |
| `Cmd-K` (macOS) / `Ctrl-K` (elsewhere)                           | Toggles the palette. Bails when typing in an input or textarea. |
| `window.dispatchEvent(new CustomEvent(SPOTLIGHT_OPEN_EVENT))`    | Opens from any component without prop drilling.                 |
| A custom button calling the above                                | Standard pattern for a "search" button in your own menu bar.    |

## Result kinds

| Kind     | Source                                                                                                          |
| -------- | --------------------------------------------------------------------------------------------------------------- |
| App      | Every entry in the `apps` prop passed to `<Desktop>` / `<DesktopProvider>`.                                     |
| System   | Every entry in the `systemWindows` registry (Settings by default, plus any you register).                        |
| External | Every result returned by a function registered via [`registerSpotlightSource`](/api/registerspotlightsource/).  |

Each row shows an accent tile, name, and tagline. Activating an App or System result calls `openWindow(...)`, the same primitive a dock click uses. External results run their own `onActivate` callback so they can navigate, dispatch, or open a URL. From "I typed three letters" to "the right thing happens" is one keystroke and one dispatch.

## Keyboard

| Key            | Effect                                                              |
| -------------- | ------------------------------------------------------------------- |
| ↑ / ↓          | Move selection. Wraps at the ends.                                  |
| Enter          | Activate the selected result.                                       |
| Escape         | Dismiss. Focus returns to whatever owned it before Spotlight opened. |
| Cmd/Ctrl-K     | Dismiss (toggle off).                                               |

## Adding your own results

Two contribution paths, depending on whether the result is a window or something else.

**Use `registerSystemWindow(...)`** when activating the result should open a window managed by react-ui-os. Settings is the canonical example.

**Use [`registerSpotlightSource(...)`](/api/registerspotlightsource/)** when activating the result should do something else: navigate to a docs page, open an external URL, dispatch a custom event, or trigger an app-specific action. A source is a function `(query) => SpotlightResult[]`, and each result owns its own `onActivate`.

```tsx
import { registerSpotlightSource } from "@react-ui-os/desktop";

useEffect(() => {
  return registerSpotlightSource("bookmarks", (query) => {
    if (!query) return [];
    return bookmarks
      .filter((b) => b.name.toLowerCase().includes(query))
      .map((b) => ({
        id: b.id,
        name: b.name,
        tagline: b.url,
        kindLabel: "Bookmark",
        onActivate: () => window.open(b.url, "_blank", "noopener"),
      }));
  });
}, []);
```

## Accessibility

- Backdrop is `role="presentation"` with click-to-dismiss.
- Panel is `role="dialog" aria-modal="true" aria-label="Spotlight"`.
- Results list is `role="listbox" aria-label="Spotlight results"`.
- Each row is `role="option" aria-selected="<bool>"`.
- Focus restoration on close — never traps the user.
