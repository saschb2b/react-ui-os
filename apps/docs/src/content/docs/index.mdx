## Why this exists

Most React UI libraries ship fifty components and let you wire them. That instinct produces tidy webapps. It does not produce a coherent OS feeling, because nobody wants to wire fifty pieces into a desktop and keep them consistent. **react-ui-os inverts the contract.** You register apps. The library composes the system. One object describing an app lights up the dock, the menu bar, Spotlight, and the keyboard shortcuts at once.

## What you get in one tag

```tsx
import { Desktop } from "@react-ui-os/desktop";
import { defaultTheme } from "@react-ui-os/theme-default";

const apps = [{ id: "notes", name: "Notes", content: () => <h1>Hello, desktop.</h1> }];

<Desktop apps={apps} theme={defaultTheme} />;
```

Wallpaper. Dock. Draggable resizable windows with macOS-style traffic lights. Focus tracking. Minimize-to-dock genie animation. Cmd-K Spotlight. Cmd-, Settings. All from that one tag.

## App Store

You do not have to build every app. Prebuilt ones live in a registry; install one with the CLI and the files land in your codebase, yours to edit.

```bash
npx @react-ui-os/cli add notes
```

[Browse the App Store](/app-store/) or [publish your own](/app-store/publish/).
