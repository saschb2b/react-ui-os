The [App Store](/app-store/) is a shadcn registry. The library ships the
registry skeleton and seeds it with the example apps; it does not bundle a
growing app suite. New apps live in registries, and the most direct way to
share one with the community is to host your own.

An app is distributed as a registry item: a small JSON file that lists the
app's source files and its npm dependencies. `npx shadcn add <url>` reads it
and copies the files into the consumer's project.

## 1. Author the app

An app is one `App` object plus whatever component tree it needs. Keep each
app in its own folder so its files copy together and the relative imports
survive the move.

```tsx title="src/os-apps/weather/index.tsx"
"use client";

import type { App } from "@react-ui-os/core";
import { useTheme } from "@react-ui-os/desktop";

function WeatherContent() {
  const theme = useTheme();
  return (
    <div style={{ color: theme.palette.textPrimary, padding: 8 }}>
      Sunny, 21°
    </div>
  );
}

export const weatherApp: App = {
  id: "weather",
  name: "Weather",
  tagline: "Today at a glance",
  accent: "#38bdf8",
  defaultBounds: { w: 360, h: 420 },
  content: WeatherContent,
};
```

Read colors, radii, and motion from `useTheme()` rather than hardcoding them,
guard any `window`/`document` access for SSR, and persist state through the
[storage adapter](/api/usewindowmanager/) instead of `localStorage` directly.
The example apps under `packages/example-apps/src` are working references for
the two-pane layout, persistence, and keyboard handling.

## 2. Describe it as a registry item

Add an entry to a `registry.json`. The schema is shadcn's; these are the
fields an app uses:

- `name`: the install id, unique within the registry.
- `type`: `registry:block` for a multi-file app.
- `title`, `description`, `categories`: shown in a gallery.
- `dependencies`: npm packages the app imports. Always
  `@react-ui-os/core` and `@react-ui-os/desktop`, plus anything else it pulls
  in.
- `files`: one entry per source file. `path` is where the file lives in the
  registry repo; `target` is where it lands in the consumer's project.
- `docs`: a post-install message. Use it to tell the consumer to register the
  app with `<Desktop>`.
- `meta`: free-form. The gallery reads `meta.accent` for the card.

```json title="registry.json"
{
  "$schema": "https://ui.shadcn.com/schema/registry.json",
  "name": "acme-apps",
  "homepage": "https://acme.com",
  "items": [
    {
      "name": "weather",
      "type": "registry:block",
      "title": "Weather",
      "description": "Today at a glance.",
      "categories": ["utilities"],
      "dependencies": ["@react-ui-os/core", "@react-ui-os/desktop"],
      "docs": "Register Weather: import { weatherApp } from \"@/components/os-apps/weather\" and add it to the apps array you pass to <Desktop apps={[...]} />.",
      "meta": { "accent": "#38bdf8" },
      "files": [
        {
          "path": "src/os-apps/weather/index.tsx",
          "type": "registry:file",
          "target": "@components/os-apps/weather/index.tsx"
        }
      ]
    }
  ]
}
```

`@components` in a `target` resolves through the consumer's `components.json`
alias, so the app lands in their components directory under `os-apps/`. Every
file of one app shares that `os-apps/<id>/` prefix, which keeps the `./icon`
and `./store` imports valid after the copy.

## 3. Build and test

`shadcn build` inlines each file's content into one JSON per item under
`public/r`:

```bash
npx shadcn@latest build
```

Install from the generated file to confirm it copies cleanly, before you
publish anything:

```bash
npx shadcn@latest add ./public/r/weather.json
```

## 4. Host your own registry

A registry is static JSON over HTTP. Serve the `public/r` directory from any
host (GitHub Pages, Vercel, an S3 bucket) and share the per-item URL:

```bash
npx shadcn@latest add https://acme.com/r/weather.json
```

Consumers who add your registry to their `components.json` can install by
name:

```json title="components.json"
{
  "registries": {
    "@acme": "https://acme.com/r/{name}.json"
  }
}
```

```bash
npx shadcn@latest add @acme/weather
```

This is the path that scales: anyone can publish a registry of react-ui-os
apps without touching this repository, and consumers pick from any of them.

## Propose an app for the official registry

The official registry is intentionally small and curated. If an app is broadly
useful and matches the design discipline in `DESIGN.md`, open a pull request
that adds its source and a `registry.json` entry pointing at it, then run
`pnpm registry:build` to confirm it builds. See the shadcn
[registry-item schema](https://ui.shadcn.com/docs/registry/registry-item-json)
for the full field reference.
